# X402 SDK 封装与仲裁工具任务

## 1. 目标
完善 `black2-sdk` 对 X402 的封装，并提供配套的仲裁测试工具。

## 🏗️ 项目架构背景（必读）
*   **SDK 结构：** 位于 `black2-sdk/black2/`，核心入口为 `client.py`。
*   **本地模式：** SDK 已支持 `local_mode=True`，用于在没有 GitHub/X402 环境下调试信誉算法。
*   **依赖管理：** 需通过 `setup.py` 管理 `uvd-x402-sdk` 依赖，确保与现有 `requests`, `pygit2` 等库兼容。

## 2. 核心任务分解

### 第一步：SDK 易用性增强
- [ ] 在 `black2-sdk/black2/x402_bridge.py` 中增加详细的 Docstring 和类型提示。
- [ ] 实现统一的错误码体系，区分“信誉拒绝”、“余额不足”和“网络超时”。
- [ ] 增加本地模式（Mock Mode），方便开发者在没有 X402 Key 的情况下调试。

### 第二步：仲裁模拟器开发
- [ ] 更新 `4/test_arbitrator.py`，增加 X402 资金流转的模拟测试。
- [ ] 场景：模拟买家申诉、仲裁员投票、X402 自动退款的全流程。

### 第三步：文档与示例代码
- [ ] 编写 `README.md`，展示如何通过 B2P SDK 一行代码发起 X402 托管支付。
- [ ] 提供 Python 和 JavaScript 的双语示例。

## 3. 预期成果
*   **开发者友好：** 外部团队可以快速理解并接入 B2P + X402 架构。
*   **鲁棒性：** 通过完善的测试用例，确保支付桥接层在各种极端情况下的稳定性。
*   **生态扩张：** 高质量的 SDK 将吸引更多 AI 开发者加入 B2P 信任联盟。
