# Black2 Clearing Protocol - 架构蓝图

**版本**: v1.0  
**更新日期**: 2026-04-20  

---

## 一、项目定位

Black2 是一个**去中心化的交易清算协议**，为 AI Agent 和人类之间的交易提供：

- **可验证**: 每笔交易都有哈希+签名证明
- **可追溯**: 交易历史通过存证机制永久保存
- **可仲裁**: 基于合同哈希和交付物哈希的自动仲裁
- **分布式信任**: 不依赖单一平台，多方独立验证

---

## 二、核心流程

```
1. 买卖双方协商合同内容
   ↓
2. 合同内容 → SHA-256 → contract_hash（双方签名确认）
   ↓
3. 创建交易（存证 contract_hash）
   ↓
4. 卖家交付 → 交付物 → SHA-256 → file_hash
   ↓
5. 自动仲裁：contract_hash == file_hash ?
   - 相等 → 卖家胜，自动放款
   - 不等 → 买家胜，自动退款
   ↓
6. 每小时锚定根哈希到 GitHub（不可篡改证据）
```

## 四、技术栈

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| **后端框架** | Python + FastAPI | 高性能异步 API |
| **数据库** | SQLite + aiosqlite | 轻量级，单机够用 |
| **加密** | SHA-256 + Ed25519 | 行业标准 |
| **存证** | GitHub API + Web3.py | 多层锚定 |
| **定时任务** | APScheduler | 每小时自动锚定 |

## 六、安全设计

### 1. 防篡改
- 每笔交易生成 SHA-256 哈希
- 用 Ed25519 私钥签名
- 每小时将根哈希存证到 GitHub/区块链

### 2. 身份验证
- 每个参与者拥有独立的 Ed25519 密钥对
- 所有 API 请求需携带签名
- 服务端验证签名有效性

### 3. 仲裁自动化
- 对比 `contract_hash` 和 `file_hash`
- 无需人工干预，代码即法律
- 仲裁结果公开可查

## 八、商业模式

### 收入来源
1. **交易手续费**: 每笔交易收取 0.5%-2%
2. **高级存证**: Arweave 永久存储服务费
3. **企业版 API**: 高并发、定制化仲裁规则

### 成本结构
- 服务器：~$5/月（DigitalOcean）
- 存证：$0（GitHub）+ $0（测试网）+ ~$10/年（Arweave）
- 总成本：< $20/月
