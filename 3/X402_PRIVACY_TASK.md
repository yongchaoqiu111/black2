# X402 隐私保护与 AI 交互规范任务

## 1. 目标
在接入 X402 的过程中，确保 AI Agent 的交易隐私符合 GDPR 等合规要求。

## 🏗️ 项目架构背景（必读）
*   **数据结构：** 信誉数据存储在 `b2p-repo.json` 中，目前包含宏观指标和微观凭证哈希。
*   **AI 交互：** 外部 AI 通过 `backend/src/api/ai_routes.py` 获取商品信息，其中已包含 `reputation_score`。
*   **隐私痛点：** X402 支付会在公链留下痕迹，需确保链上地址与 AI 的真实身份（Git 仓库所有者）解耦。

## 2. 核心任务分解

### 第一步：数据脱敏与选择性披露
- [ ] 设计 `b2p-repo.json` 的隐私字段：仅公开宏观指标（信誉分），隐藏微观细节。
- [ ] 在调用 X402 支付时，使用一次性地址或代理身份，避免暴露 AI 背后的真实钱包。

### 第二步：零知识证明 (ZKP) 预研
- [ ] 调研是否需要在 B2P SDK 中集成 ZKP，以证明“我有足够信誉”而不泄露具体交易历史。
- [ ] 设计隐私保护的 API 接口，供外部 AI 进行合规性检查。

### 第三步：AI 交互标准化
- [ ] 更新 `AI_AGENTS.md`，定义 AI 如何解析 X402 返回的支付指令。
- [ ] 明确 AI 在收到“资金已锁定”信号后的行为逻辑（开始交付数据/服务）。

## 3. 预期成果
*   **合规性：** 交易流程满足数据最小化原则，保护用户隐私。
*   **身份匿名：** 外部观察者无法通过 X402 链上数据反推 AI 的真实商业关系。
*   **标准化交互：** 为成千上万个 AI Agent 提供统一的隐私保护接入指南。
