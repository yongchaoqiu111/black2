## 4 周开发计划

### 第 1 周：准备阶段（搭好基础服务）

**目标**: 清算协议 v1.0 能跑起来

| 天数 | 任务 | 具体内容 | 产出 |
|------|------|----------|------|
| 第 1 天 | 搭环境 | 创建项目结构，安装依赖 | 目录就绪 |
| 第 2 天 | 写哈希 + 签名模块 | SHA-256 哈希、Ed25519 签名验证 | `crypto/` 模块 |
| 第 3 天 | 写交易记录模块 | 设计交易数据结构（from, to, amount, contract_hash, file_hash, timestamp） | `db/` 模块 |
| 第 4 天 | 写 API 接口 | 创建交易、查询交易、验证交易 | `api/` 模块 |
| 第 5 天 | 写定期锚定脚本 | 每小时把交易根哈希发到公共平台 | `anchor/` 模块 |
| 第 6-7 天 | 测试 + 修复 | 本地测试所有接口，修复 bug | 可用的 API v1.0 |

**验收标准**:
- [ ] 能通过 API 创建一笔交易，返回哈希 + 签名
- [ ] 能通过 API 验证一笔交易是否真实
- [ ] 锚定脚本能正常运行

---

### 第 2 周：核心阶段（跑通 AI→AI 交易）

**目标**: 两个 AI 脚本在你的协议上完成第一笔交易

| 天数 | 任务 | 具体内容 | 产出 |
|------|------|----------|------|
| 第 1-2 天 | 写 AI 买家脚本 | 自动发布"买一篇 100 字文章，价格 100 元"的需求 | `buyer_agent.py` |
| 第 3-4 天 | 写 AI 卖家脚本 | 自动接单、生成文章、提交、标记完成 | `seller_agent.py` |
| 第 5 天 | 集成清算 API | 两个脚本在交易过程中调用清算 API 记录每一步 | 完整交易流程 |
| 第 6-7 天 | 跑通 + 记录 | 运行脚本，完成第一笔 AI→AI 交易，截图保存 | 第一笔交易记录 |

**验收标准**:
- [ ] AI 买家脚本能自动发布需求
- [ ] AI 卖家脚本能自动接单并完成
- [ ] 交易过程中清算 API 被成功调用
- [ ] 有一张"交易成功"的截图/日志

---

### 第 3 周：验证阶段（跑通仲裁闭环）

**目标**: 制造一次纠纷，用协议验证并仲裁

| 天数 | 任务 | 具体内容 | 产出 |
|------|------|----------|------|
| 第 1-2 天 | 修改卖家脚本 | 让卖家脚本故意提交"不符合要求的文章" | 故障注入 |
| 第 3 天 | 跑出纠纷 | 运行买卖双方脚本，买家拒付，纠纷产生 | 纠纷日志 |
| 第 4-5 天 | 实现仲裁逻辑 | 写仲裁函数：对比合同哈希和交付物哈希 | `arbitrator.py` |
| 第 6 天 | 仲裁验证 | 对纠纷进行仲裁，输出结果 | 仲裁记录 |
| 第 7 天 | 记录案例 | 把整个纠纷 + 仲裁过程写成文档 | 第一个仲裁案例 |

**验收标准**:
- [ ] 能制造一个真实的纠纷场景
- [ ] 仲裁函数能正确判断谁违约
- [ ] 有一个完整的"纠纷→仲裁"案例文档

---

### 第 4 周：展示阶段（对外展示 + 找用户）

**目标**: 让全世界看到成果，找到第一个外部用户

| 天数 | 任务 | 具体内容 | 产出 |
|------|------|----------|------|
| 第 1-2 天 | 录视频 | 3-5 分钟屏幕录制：展示 AI 交易→纠纷→仲裁全过程 | 演示视频 |
| 第 3 天 | 写案例文档 | 把第 3 周的案例写成正式文档（含流程图、代码片段、仲裁逻辑） | `CASE_STUDY.md` |
| 第 4 天 | 开源代码 | 把清算 API、买卖脚本、仲裁逻辑推送到 GitHub | GitHub 仓库 |
| 第 5 天 | 写协议规范 | 写一份简洁的协议规范（1-2 页），定义数据结构、API 格式、仲裁规则 | `PROTOCOL.md` |
| 第 6 天 | 发布 | 视频发 YouTube/B 站，文档发 Twitter/V2EX/Hacker News | 公开亮相 |
| 第 7 天 | 找第一个外部用户 | 私信 3-5 个做 AI Agent 的开发者，问他们要不要接入 | 一个意向用户 |

**验收标准**:
- [ ] 视频已发布，至少有一个平台有播放
- [ ] GitHub 仓库已开源，有 README 和代码
- [ ] 发出了至少 3 条"找用户"的私信/邮件

## 技术选型

| 组件 | 选择 | 原因 |
|------|------|------|
| 后端 | Python + FastAPI | 最快上手，哈希库现成 |
| 数据库 | SQLite | 单机够用，无需额外配置 |
| 哈希算法 | SHA-256 | 标准，无需自己实现 |
| 签名算法 | Ed25519 | 简单、安全、库现成 |
| 锚定平台 | GitHub Gist + 时间戳 | 免费、有记录、不可篡改 |
